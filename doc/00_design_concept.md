# 小説データクレンジング・評価システム 設計コンセプト

## 1. 概要
本システムは、CSV形式の小説データを唯一のソースとし、機械学習（LLMファインチューニング）に適した高品質なデータセットを構築することを目的としています。
「おかしな日本語」の排除と、「文章の良し悪し」の定量的評価を行い、閾値（足切り）に基づいてデータをフィルタリングします。

## 2. 評価の軸：一つの指標による統合
複雑な多角的評価ではなく、**「言語モデルとしての自然さ（Perplexity: 困惑度）」**を主軸とした単一の評価スコアを採用します。
これにより、直感的に「良い文章（低いPP）」「悪い文章（高いPP）」を判別できます。

### 評価パイプライン
1.  **Rule-based Filter (事前篩い)**: 明らかに異常なデータをコストをかけずに弾く。
2.  **Model-based Scoring (本評価)**: 軽量言語モデルを用いて困惑度（Perplexity）を計算し、スコア化する。
3.  **Thresholding (足切り)**: スコアが閾値を超える（＝質が悪い）データを除外する。

## 3. 各工程の詳細

### 3.1. Rule-based Filter (おかしな日本語の排除)
計算コストのかかるAIモデルを通す前に、明確なルールでノイズを除去します。
- **文字種比率の異常検知**: 漢字が多すぎる、ひらがなが少なすぎる、記号や英数字が異常に多い行を排除。
- **長さの制限**: 短すぎて文脈がないもの、長すぎて切れているものを調整・除外。
- **特殊文字の検出**: 制御文字や意味のない連続記号を含むものの排除。

### 3.2. Model-based Scoring (品質評価と重み付け)
事前学習済みの軽量日本語モデル（例: `rinna/japanese-gpt2-small` など）を使用し、各行のテキストに対する**Perplexity (PP)** を計算します。
- **Perplexityとは**: モデルがその文章を予測するのにどれくらい「驚いたか」を示す指標。
  - **PPが低い** = モデルにとって自然な文章 = **良い文章**
  - **PPが高い** = モデルにとって不自然な文章 = **悪い文章**
- このPP値自体を「品質スコア（逆数）」として扱い、重み付けの基準とします。

### 3.3. Thresholding (足切り)
計算されたPP値の分布を確認し、上位X%（または絶対値閾値）を「悪い文章」とみなして削除します。
- 例: PP値が高い順にソートし、下位80%のみを採用するなど。

## 4. 実行環境戦略 (Local vs Cloud)
- **データ前処理・評価 (Local/Colab)**:
  - 16GB RAMのAMD環境でも動作するよう、バッチ処理を実装。
  - 大規模なモデルは使用せず、小規模なGPT-2等を使用することでGPUなしでも推論可能にする（時間はかかるが確実）。
  - Colab/Kaggle利用時はGPUを活用し高速化。
- **モデル学習 (Colab/Kaggle推奨)**:
  - クレンジング済みのデータを用いて、実際にLLMをファインチューニングする工程は、GPU環境であるColabやKaggleでの実行を前提とする。

## 5. 出力物
1.  **`cleaned_dataset.csv`**: フィルタリング後の高品質データ。
2.  **`scored_dataset.csv`**: 全データにPPスコアを付与したもの（分析用）。
3.  **`filter_report.txt`**: どの基準で何件除外されたかのログ。

## 6. 今後の拡張性
- 現在は「一つの軸（PP）」だが、必要に応じて「話題の一貫性」や「感情スコア」などを加算して複合スコア化することも可能。
- 足切り閾値はパラメータとして外部から調整可能にする。

## Phase 2: 新奇性・意外性の発掘 (Novelty & Clustering)
**目的**: 
人間が通常書かないような「意外な組み合わせ」や「叙述の型」を発見し、データの偏りを可視化する。
単に「正しい日本語」ではなく、「人間には思いつかないが、文脈として成立し、面白い」領域を特定する。

**処理内容**:
1.  **意味空間への埋め込み**: クリーン化したデータをベクトル化（Embedding）。
2.  **グルーピング（クラスタリング）**: 類似した叙述パターンを自動分類。
3.  **新奇性スコアリング**: 
    *   クラスタ中心からの距離が遠いもの（外れ値）を「意外な組み合わせ」として検出。
    *   特定のクラスタに偏りすぎているものを検知し、多様性を確保するための重み付け材料とする。
4.  **「通常でない」ものの認知**: 統計的に稀だが、Perplexity（困惑度）が高すぎない（＝破綻していない）ものを「良質な新奇性」としてフラグ付け。

**出力**:
- `clustered_dataset.csv`: クラスタIDと新奇性スコアが付与されたデータ。
- `novelty_report.txt`: 発見された「意外な組み合わせ」のサンプルと、分布の偏りレポート。

### 🛠 スタック (Stack)
- **SentenceTransformers**: 文章の意味ベクトル化 (`cl-tohoku/bert-base-japanese-v2` など)。
- **Scikit-learn / UMAP / HDBSCAN**: 次元削減とクラスタリング。
- **NumPy/Pandas**: 統計計算。

### 📚 理論・概念・用語メモ
- **Semantic Embedding (意味埋め込み)**: 単語や文章を連続的なベクトル空間に変換し、意味的な近さを距離で表現する技術。
- **Clustering (クラスタリング)**: データを類似性に基づいてグループ分けする手法。ここでは「叙述の型」を自動発見するために使用。
- **Outlier Detection (外れ値検出)**: 分布から大きく外れたデータポイントを検出すること。本プロジェクトでは「人間が通常書かない表現」の候補とする。
- **Novelty vs Nonsense**: 
    - `Nonsense`: 文法破綻や意味不通（Phase 1 で排除済み）。
    - `Novelty`: 文法的には正しいが、共起確率が低く意外性があるもの（Phase 2 で抽出）。
- **Diversity Sampling**: 学習データに特定のタイプばかり含まれるとモデルが偏るため、多様なクラスタからバランスよくサンプリングする手法。

## Phase 3: 創造的再生産 (Creative Generation)
**目的**: 
Phase 1, 2 で選別・評価された「高品質かつ新奇性のあるデータ」を用いてモデルを学習させ、人間には書けないが面白い文章を生成する。

**処理内容**:
1.  **データサンプリング**: 新奇性スコアに基づき、偏りを抑えつつ意外性の高いデータを選択。
2.  **ファインチューニング**: 小規模 LLM に対し、選択されたデータで学習。
3.  **生成的評価**: 生成された文章が、学習データの「新奇性」をどの程度継承しているかを評価。

### 🛠 スタック (Stack)
- **Hugging Face Transformers / PEFT (LoRA)**: 効率的なファインチューニング。
- **Accelerate**: リソース最適化。

### 📚 理論・概念・用語メモ
- **LoRA (Low-Rank Adaptation)**: モデルの重みの一部のみを更新することで、少ないリソースで効率的にファインチューニングを行う手法。
- **Temperature Sampling**: 生成時の確率分布の鋭さを調整し、意外性（ランダム性）をコントロールするパラメータ。
- **Mode Collapse**: 生成モデルが多様性を失い、同じような出力しか出さなくなる現象。これを防ぐために Phase 2 のグルーピングが重要となる。
